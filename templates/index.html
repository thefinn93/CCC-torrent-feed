I've a quick script to hit CCC Atom feed and add .torrent to the links, so they can be loaded into a torrent client or whatever. Here's the URLs n stuff: {% for feed in feeds %} {% endfor %}
FeedURLicon
{{ feed }} {{ url_for('feed', name=feed, _external=True) }}
Note: ruTorrent inexplicably prepends the torrent URLs with the feed URL, causing it to break. I worked around this with a rule in the Rules Manager (in the context menu of every RSS feed). Create a new rule with the following configuration:
  • If the URL fo the torrent description matches pattern: |.*cdn.media.ccc.de/(.*)|i
  • then replace the URL of the torrent download with: https://cdn.media.ccc.de/${1}
I've read through the ruTorrent RSS code a bit, but haven't figured out why this happens. If anyone has any insight, please file an issue
